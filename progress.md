# Provenance Guard — Progress Summary

## Project Overview

Provenance Guard is a Flask backend that creative platforms can plug into to
classify whether submitted text was written by a human or generated by AI.
It returns a confidence score, a plain-language transparency label, and logs
every decision to an audit trail. Creators can contest classifications via an
appeals workflow

## Detection Signals

### Signal 1 — LLM Classifier (`detector.py`)

Sends the text to Groq (`llama-3.3-70b-versatile`) with a prompt asking for a
`verdict` (`"human"` or `"ai"`) and a `raw_confidence` (0.0–1.0).

**Score direction:** `llm_score = raw_confidence` if AI; `1.0 - raw_confidence`
if human. So 0.0 always means confident human, 1.0 always means confident AI.

**Why it works:** Captures semantic and holistic properties — tonal consistency,
idiosyncratic phrasing, whether the voice feels personal or optimized. Prompt
uses low temperature (0.1) for consistent output. Parser strips markdown fences
and clamps confidence to [0, 1] for robustness.

**Blind spots:** Short texts (< 100 words), clean structured human writers,
lightly edited AI output, genre-specific prose where uniformity is a convention.

---

### Signal 2 — Stylometric Analyzer (`detector_stylo.py`)

Pure Python — no external libraries. Computes four metrics and averages their
per-metric scores into a single `stylo_score`:

| Metric | Human-like | AI-like | Threshold (human → AI) |
|---|---|---|---|
| Sentence length variance (std dev) | High (≥ 30) | Low (≤ 5) | Inverted |
| Type-token ratio (unique/total words) | High (≥ 0.90) | Low (≤ 0.60) | Inverted |
| Punctuation density (per 100 words) | High (≥ 8) | Low (≤ 2) | Inverted |
| Average word length (chars) | Short (≤ 4) | Long (≥ 6.5) | Direct |

**Short-text dampening:** Texts under 50 words have their score pulled toward
0.5 proportionally, because variance metrics are meaningless with only 2–3
sentences. The response includes a `stylo_warning` field when this applies.

**Blind spots:** Genre confound (legal/academic writing looks AI-like), texts
deliberately stylized to be irregular, anything under 50 words.

## Confidence Scoring

**Formula (from planning.md):**
```
ai_probability = (0.60 × llm_score) + (0.40 × stylo_score)
```

LLM gets 60% weight because it captures semantic patterns the stylometric
signal misses entirely. Stylometrics get 40% as a structural cross-check.

**Thresholds:**

| ai_probability | classification | confidence_level |
|---|---|---|
| ≥ 0.80 | `ai` | `high` |
| ≤ 0.20 | `human` | `high` |
| 0.21–0.79 | `uncertain` | `uncertain` |

The thresholds are intentionally asymmetric — a false positive (calling a human
AI) is worse than a false negative, so the system only calls `ai` at ≥ 0.80.

## Transparency Labels

Three variants, all returned as the `label_text` field in `/submit` responses:

**High-confidence AI** (`ai_probability ≥ 0.80`)
```
⚠️ AI-Generated Content
Our analysis strongly suggests this work was generated by an AI writing tool, not
written by the credited creator. Confidence: High. If you are the creator and believe
this is incorrect, you may contest this classification using the appeals process.
```

**High-confidence Human** (`ai_probability ≤ 0.20`)
```
✅ Human-Authored
Our analysis strongly suggests this work was written by the credited creator.
Confidence: High.
```

**Uncertain** (`0.21 ≤ ai_probability ≤ 0.79`)
```
🔍 Authorship Uncertain
Our system could not confidently determine whether this work was written by a human
or generated by AI. This may reflect an unusual writing style, a short submission,
or signals that disagree. The creator may contest this classification. This label
may be updated following a review.
```

## Key Decisions & Rationale

**Why Groq at 60% weight, not 50/50?**
The LLM reads holistic semantic patterns — tone, voice, whether the text feels
like it's reaching for something. The stylometric signal only sees structure.
On borderline cases (formal human writing, lightly edited AI), the LLM is more
informative, so it earns more weight.

**Why 0.80 threshold for "high-confidence AI", not 0.70?**
False positives (mislabeling human work as AI) are reputational harm to real
people. The system is designed to be conservative — it would rather say
"uncertain" than accuse. 0.80 means both signals have to lean AI fairly hard
before the system commits.

**Why short-text dampening in stylometrics?**
Variance is meaningless with 2 sentences. A 30-word haiku with no sentence
variation would score very high on the variance metric purely because it's short,
not because it's AI-like. Pulling toward 0.5 below 50 words is more honest than
a false high-confidence score.

**Why JSONL for the audit log?**
Append-only, human-readable, no setup, easy to tail in the terminal, and every
line is independently parseable. SQLite would be more queryable but adds
infrastructure complexity for no grading benefit.

## Bugs Found & Fixed During Development

| Issue | Root cause | Fix |
|---|---|---|
| `GET /` returning 404 | Hitting root URL, not a valid route | Directed to `/submit` and `/log` |
| `400` on valid curl | Missing `-H "Content-Type: application/json"` | Added header |
| `400` persisting with header | Windows PowerShell mangling single-quoted JSON | Switched to `Invoke-WebRequest` with `@{...}` headers |
| `ModuleNotFoundError: detector` | File renamed locally to `detector.py` but import still said `detector_llm` | Updated import in `app.py` |
| Debug prints not appearing | Local repo running different `app.py` than the sandbox copy | Confirmed path with `__file__` print |
| TTR scores all 0.0 on test inputs | Thresholds set too high (0.75/0.40) for realistic short texts | Recalibrated to 0.90/0.60 |
